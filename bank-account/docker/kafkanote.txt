kafka broker is a server running in a kafka cluster (container)
brokers are stateless and relies on zookeeper
zookeeper is responsible for manage cluster
for reliability minimum 3 brokers needed

topics are divide into partitions in a kafka cluster
partitions are replicate around brokers

a topic can be viewed as a channel though which event data is streamed,
producers always send or produce messages to a topic, while consumers consume events from topics that they subscribe to.
